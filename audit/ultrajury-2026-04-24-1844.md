## Problèmes critiques (action immédiate)

1. **`PrivacyInfo.xcprivacy` absent** → risque de rejet Apple sur la v1.1 Build 3 en review.
2. **Mentions légales LCEN non conformes** (amende jusqu'à 75 000 €) + `/privacy` dit "aucune donnée transmise" alors que `/api/feedback` envoie à Resend → **fausse déclaration RGPD** art. 13.
3. **HTML injection email + pas de rate limit `/api/feedback`** → phishing-relay via Resend + épuisement quota 100 emails/jour.
4. **Homepage `noindex` parasite** → app complètement invisible à Google.
5. **Android keystore non gitignoré** (`chronos-android/android.keystore` + `.aab` + `.apk`) — dès le prochain `git add -A`, clé de signature Play Store fuite.
6. **Aucun CI/CD, aucun monitoring, aucun rollback** → panne prod détectée par l'utilisateur.

## Plan d'action

### Semaine 1 (quick wins sécurité + compliance)
- Ajouter `PrivacyInfo.xcprivacy` + corriger `/cgu` + `/privacy` (#1 #2)
- Escape HTML complet + rate limit + schéma Zod feedback (#3)
- `npm audit fix` + upgrade next ≥ 16.2.3 (#4)
- Compléter `.gitignore` : `*.keystore`, `*.aab`, `*.apk`, `ios/App/App/public/`
- HSTS + CSP (#7)

### Semaines 2-3 (SEO + DevOps structurels)
- Landing SSR `/` + JSON-LD + OG + `metadataBase` (#5 #8)
- `app/robots.ts` + `app/sitemap.ts`
- CI/CD GitHub Actions + `/api/health` + UptimeRobot (#6)
- Releases versionnées `/var/www/chronos/releases/vN` + symlink pour rollback

### Mois 2+ (qualité + UX)
- Nettoyage charte (indigo → bleu) + prefers-reduced-motion (#9)
- Suppression dead code + version sync (#10)
- Split `calculations.ts` en modules + service layer `lib/services/feedback.ts`
- Tests composants/hooks (coverage global 40% → 70%)

## Résumé exécutif

Chronos est un produit utilitaire **techniquement mature** (architecture propre, TS strict, tests métier solides, PWA soignée, contenu francophone irréprochable) mais **fragile sur ses frontières** : conformité RGPD + Apple Privacy Manifest absents, SEO neutralisé par une redirection buggée, déploiement manuel sans filet, et une seule API exposée cumule trois vulnérabilités évitables.

**Score 63.2 / 100** — une semaine de corrections réglementaires et sécuritaires (#1→#4) remonte au-delà de 70 sans toucher au code métier.
